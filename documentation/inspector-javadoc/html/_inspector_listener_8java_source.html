35  * Single entry from JPF inspector.
36  * Listener dynamically forwards all methods to specified listener instance.
37  * Class can dynamically change listener to which methods are forwarded and thus
38  * implement different modes of work.
39  * Modes are stacked, used mode is in the top of the modes stacks (others are not notified)
40  *
41  * Currently modes of works are supported
42  * {@link InspectorListenerModeNotifications} - Default mode when whole Inspector is notified about all interesting events
43  * {@link InspectorListenerModeSilent} - Silent mode used in backward steps, when only command listener is notified.
