boolean gov.nasa.jpf.inspector.server.breakpoints.CommandsManager.run = true
private

If true, then the inspector should run and continue execution.

If false, it should stop on next step because of a user command (not breakpoint). Calling "break" is the primary way to set this variable to false.

Note: The Inspector thread sets this variable, the JPF thread reads this variable. It's not really thread-safe, but we only use it to break a running program and don't care about a few instructions here or there and also the user is unlikely to type commands quickly enough for it to be a problem.
