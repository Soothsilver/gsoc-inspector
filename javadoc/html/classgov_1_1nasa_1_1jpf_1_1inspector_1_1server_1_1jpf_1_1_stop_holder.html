Detailed Description

Handles resuming and stopping JPF execution.

This is a very important class central to the server's operation.

This class also holds current state of the SuT (JVM).

Implementors: When modifying this class, make sure to think about thread safety and synchronization.

boolean gov.nasa.jpf.inspector.server.jpf.StopHolder.preventJpfFromResuming ( )

Primary documentation at JPFInspectorBackEndInterface#preventJpfFromResuming().

Local copy of documentation:

If JPF is currently blocked, then it will be prevented from resuming until permitJpfToResumeAgain() is called. If JPF is currently running, this method will simply return false.

If multiple calls to this method are made, then all of them must be undone using permitJpfToResumeAgain() before JPF can resume again.

Synchronized on the access lock.

Returns
True if JPF is currently stopped and will be prevented from resuming; false otherwise.

final ReentrantLock gov.nasa.jpf.inspector.server.jpf.StopHolder.accessLock = new ReentrantLock()
private

The access lock is used to synchronize modifications to this StopHolder object.

The access lock must never be held by a thread for a long time. It is used to ensure simplicity and methods should use it to ensure that only a single method modifies a StopHolder's property at a time.
