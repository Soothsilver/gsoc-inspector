Detailed Description

Handles resuming and stopping JPF execution.

This class also holds current state of the SuT (JVM).

Implementors: When modifying this class, make sure to think about thread safety and synchronization.

synchronized void gov.nasa.jpf.inspector.server.jpf.StopHolder.resumeExecution ( )

The only blocked thread should be the JPF thread in the stopExecution(InspectorState) method.

Threads which calls waitUntilStopped() a) before stopExecution(InspectorState) are notified in stopExecution(InspectorState) b) after stopExecution(InspectorState) are not blocked and pass through waitUntilStopped() method.

void gov.nasa.jpf.inspector.server.jpf.StopHolder.stopExecution ( InspectorState  inspState)

Notifies the client that JPF is now stopped and then blocks execution until it is woken up by a notify call.

Executed by the JPF thread.

Note: To prevent deadlocks cannot hold any lock if this methods is called.
