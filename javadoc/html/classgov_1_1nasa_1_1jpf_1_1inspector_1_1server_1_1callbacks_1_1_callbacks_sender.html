Detailed Description

Class used to handle clientCallbacks.

This class serializes clientCallbacks in the correct order and can postpone JPF stopped clientCallbacks.

This class should handle all clientCallbacks which should occur after JPF is stopped. (Prevent creation of separate threads for each clientCallbacks and related non-determinism in clientCallbacks order due to scheduling)

final List<CallbackCommand> gov.nasa.jpf.inspector.server.callbacks.CallbacksSender.callbackQueue
private

Queue of clientCallbacks scheduled to be sent from the server to the client.

This queue also acts as the monitor mutex for this class and as the condition variable for this class.

The queue itself is created using Collections#synchronizedList(List) which seems excessive because we are doing locking manually anyway, but I guess a second failsafe does not hurt.
