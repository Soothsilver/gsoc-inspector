Detailed Description

Contains loaded configuration relevant to JPF Inspector.

While we want to avoid singleton classes accessed statically in general, in this case, since JPF Inspector should be quite configurable, we would need to pass an instance of this class everywhere or at least to the panel, the client and the server and then that's pretty much the same thing as static access, and would be more messy.
