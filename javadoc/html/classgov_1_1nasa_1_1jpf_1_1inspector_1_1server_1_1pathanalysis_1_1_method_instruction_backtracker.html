Detailed Description

This backtracker has the capacity to keep track of what method and line of code we are in, which is useful for the commands back_step_over, back_step_in and back_step_out.

Initializes a new instance of MethodInstructionBacktracker.

Internally, this creates a new TransitionThreadBacktracker using the path and the current thread and a new StepThreadBacktracker to iterate back through our transition path. This backtracker is initialized to the current step of the current transition.

Parameters
pathThe current transition path, including the current transition (i.e. after a call to VM#updatePath()).

Step gov.nasa.jpf.inspector.server.pathanalysis.MethodInstructionBacktracker.backtracker ( int  howManyMethodsToLeave)
private

Returns the first step we encounter during backtracking, but all steps we encounter before we exit the specified number of methods from the call stack are ignored.

Returns null if no such step exists (for example, because we are in the first step of the thread).

Note: Use the argument "0" to backtrack to an instruction within the current method (back_step_over).

Note: Use the argument "1" to leave the current method and find the instruction in the parent (back_step_out).

Parameters
howManyMethodsToLeaveHow many methods we must exit before we return a step.
Returns
First step we encounter during backtracking that is in the method after we exit the specified number of methods, or null if no such step exists.

Returns the number of steps in this transition, prior to the specified step and including the specified step, are associated with the same instruction as the specified step.

The specific instruction only is considered, not all instructions with the same mnemonic.

Useful in BackwardBreakpointCreator to create an instance of ExpressionBreakpointInstruction.

Parameters
transitionTransition to search through.
stepWe are interested in steps strictly before this step.
Returns
The number of times the step's instruction is executed before the step, plus +1 for its execution during the step. Thus, the return value is always 1 or more.
